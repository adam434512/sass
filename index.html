sass introduction

sass stands for syntactically awesome stylsheets

sassis a styleesheet language that is compiled to css sass has powerful and magical features like mixin, function, nested rule etc

sass support two types of syntax. so its up to you and your team to choose up

  • scss syntax: which contains full set of css syntax with few exceptions.
  • indentedd synstanx: its the original sass syntanx.

variables

sass variables are variables defined and assigned the values which can be re-used throughout the stylesheet. variable make easier to change the value of the variables and also reduce repitition

mixins and include

mixins allows a developer to write a style that can be re-use throughout the stylesheet. mixins are include in the current context using the @include rule with the name of the mixin included.

sass @error rule

when writing a mixins and functions that accepts arguments developer wants to make sure those arguments have type and formats your api expects. if not the developer need to be notified and the respective mixins or function needs to stop running

sass flow control

sass provides a number of at-rules that make it possible to control whether style emit, or emmit them multiple time with small variations. Flow control can also be used inside mixin and function to write small algorithim to make writting your sass easier. sass has four flow control rules described as follows.

@if and @else

control flow that control weather or not its block will be evealuated

@each

@each rule make it easier to emit or evaluate code for each element in a list or for each value in a map. it work great for repetitive style that only has few variations between them. The block is evaluated for each element of the list in turn, which is assigned to the given variable name

sass values

sass supports number of value types, most of which comes straight from css Every expression produce a value, variables hold values. Most values comes straight from css are as follows

  • numbers. which may or may have no units eg. 17px or 17
  • string. which may or may have no quotes
  • colors. which can be reffered by their names, hex values, rgb and rgba values or hsl and they can returned from the function as well.
  • list of values. which may be separated by comma or space and which may be enclosed in square brackets or no brackets at all.
  • booleat value. true and false.
  • singleton value null
  • maps. which associates values with keys
  • function reference. returned by get-function() and call by call().

sass built-in module

sass provides many built-in modules which contain useful functions ( and the ocasional mixin). These module can be loaded with @use rule like any other user defined stylesheet, and their function can be called like any other module member. All in built module begin with sass: to indicate that they are part of sass it self

sass support the following modules

  • sass:math. provides a function that operate on numbers
  • sass:string. module make it easier to combine, search, or split apart string.
  • sass:color. generates new color based on the existing ones. making it easier to build color themes.
  • sass:list. allows a developer to access and modify values in lists.
  • sass:map. enable developer to look up the value associated with the key in a map.
  • sass:selector. allow a developer to access a powerful sass selector engine.
  • sass:meta. expose the details of sass inner working.
